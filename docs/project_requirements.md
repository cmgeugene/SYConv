# 프로젝트 요구사항 및 기술 스택 논의 (Project Requirements & Architecture)

## 1. 프로젝트 목표
이미지 또는 PDF 형태의 영어 문제(2열 구성) 문서를 입력받아, 문제 단위로 청킹하고 형광펜 처리된 단어를 추출하여 분석한 뒤, 사용자가 검수하고 최종적으로 XML(또는 엑셀) 보조 포맷으로 추출하는 애플리케이션 개발.

## 2. 주요 기능 및 요구사항
1. **문서 입력 및 청킹**
   * 이미지 및 PDF 파일 업로드 지원.
   * 2열 문서 레이아웃 분석 및 문제 단위(지문/본문/선택지) 청킹.
2. **단어 및 형광펜 마스킹 추출**
   * 본문과 선택지 내에서 형광펜으로 하이라이트된 단어를 인식하여 추출.
   * 추출된 단어의 품사, 뜻, 원형 식별 (관용구 포함, 동사는 원형 표기 원칙).
3. **사용자 검수 UI (좌우 2분할 레이아웃)**
   * **좌측 UI (원본 뷰어):** 문서 원본에 청킹된 문제 영역과 인식된 하이라이트 단어를 본문 위에 시각적(마스킹 박스 등)으로 표시.
   * **우측 UI (데이터 시트):** 엑셀/시트 스타일의 테이블로 컬럼(단어 / 품사 / 뜻) 구성. 문서 인식이 정확하지 않을 때를 대비해 사용자가 직접 시트를 편집할 수 있는 기능 지원.
4. **결과물 출력**
   * 검수가 완료된 데이터를 XML(또는 호환 가능한 엑셀 파일 등) 형식으로 다운로드.

## 3. 기술 스택 검토 (하이브리드 방식 채택)
완전 로컬 방식(Ollama + EasyOCR)은 VRAM 8GB 환경에서 메모리 스왑 및 성능 저하 우려가 크며 처리 속도가 현저히 느려질 수 있습니다. 따라서 **비용 효율, 사용자 경험(빠른 속도), 자연어 분석의 정확도가 압도적으로 높은 하이브리드 방식**을 채택합니다.

### 3.1. 로컬 환경 (이미지 분석 및 UI 구동)
* **역할:** 이미지 전처리, 영역 분석(청킹), 형광펜 색상 인식, 텍스트(OCR) 읽기, 앱 동작.
* **백엔드 기술 (Python):**
  * `FastAPI`: 로컬 웹 서버 및 프론트엔드 통신 API.
  * `OpenCV`: 이미지 처리, 2열 레이아웃 분석 및 형광펜 색상 범위(HSV) 박스 좌표 추출.
  * `EasyOCR` / `Tesseract` / `Surya-OCR` 등: 가벼운 리소스로 텍스트 및 좌표 추출.
* **프론트엔드 기술 (Web 기반):**
  * `React`, `Next.js` 등 웹 기술 기반 UI 구축 (필요시 Tauri 또는 Electron을 통해 설치형 데스크톱 앱으로 전환).
  * 캔버스(Canvas) 라이브러리: 원본 이미지 위에 추출된 박스 좌표를 오버레이로 렌더링.
  * Ag-Grid 또는 React-Data-Grid: 우측 데이터 시트(편집 가능) 구현.

### 3.2. 클라우드 API (LLM 기반 자연어 파싱)
* **역할:** 추출된 단어들의 문맥 파악, 관용구 인지, 품사 및 뜻/원형 식별 등 무거운 NLP 작업.
* **기술:**
  * `OpenAI API (gpt-4o-mini)` 또는 `Anthropic API (claude-3-haiku)`: 매우 저렴한 텍스트 기반 비용으로 수 초 내에 완벽한 자연어 처리 결과 반환.
  * 프롬프팅 기법 적용: 전처리된 텍스트와 하이라이트 좌표 정보를 보내 "문맥 안에서 이 특정 단어(관용구)들의 원형/품사/한국어 뜻을 추출해서 JSON으로 반환하라"고 지시.

## 4. 향후 개발 마일스톤 (제안)
* **Step 1 (핵심 로직 PoC):** 샘플 PDF/이미지를 파이썬 스크립트에 넣고, OpenCV 기반으로 2열 청킹 및 형광펜 박스 추출, 그리고 OCR 텍스트가 정확히 매칭되는지 콘솔 형태로 확인.
* **Step 2 (LLM API 파싱):** 로컬에서 추출한 텍스트를 LLM API로 전송하여, 원하는 형태(단어/품사/뜻)의 완벽한 JSON 응답을 받아오는지 연동 테스트.
* **Step 3 (UI 뼈대 및 연결):** React를 사용하여 좌(뷰어)/우(시트) 레이아웃을 만들고, 추출 결과 데이터를 화면에 보여주고 수정 가능하도록 양방향 바인딩.
* **Step 4 (패키징 및 완성):** XML 출력 및 파일 다운로드 기능, 최종 완성 단계에서의 로컬 앱 통합(실행 스크립트 등) 마무리.
